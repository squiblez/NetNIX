        REINSTALL(1)              NetNIX Manual              REINSTALL(1)

        NAME
            reinstall - restore factory binaries, libraries, and man pages

        SYNOPSIS
            reinstall [--ask-factory | --keep-factory | --no-factory]
            reinstall -h | --help

        DESCRIPTION
            reinstall is a system maintenance command that overwrites the
            built-in NetNIX commands, libraries, and documentation with
            the factory versions shipped inside the NetNIX executable.

            This is useful when:
            * You have updated NetNIX and want to install new commands
            * A built-in command or library has been accidentally modified
            * Files in /bin, /sbin, or /lib have been deleted or corrupted
            * You want to restore the default manual pages
            * You want to reset a factory config (e.g. /etc/httpd.conf,
              /etc/telnetd.conf, /etc/nxagent.conf) back to its shipped
              default after experimenting with it

            By default reinstall now OVERWRITES every factory file it
            finds, including configs in /etc. This is the behaviour you
            usually want after a NetNIX update. To preserve customised
            configs, use --keep-factory; to be prompted per-file, use
            --ask-factory; to skip the factory-files step entirely, use
            --no-factory.

        OPTIONS
            --ask-factory    For each factory file that already exists in
                             the VFS, prompt:
                                 (y) overwrite this one
                                 (n) ignore this one
                                 (a) overwrite all remaining
                                 (q) ignore all remaining
                             Useful when most factory files should be
                             refreshed but a couple have local edits.

            --keep-factory   Reinstall everything but skip any factory
                             file that already exists. Equivalent to the
                             pre-fix legacy behaviour.

            --no-factory     Skip the entire factory-files step. Binaries,
            --skip-factory   libraries, and man pages are still restored,
                             but nothing under /etc (or any other Factory/
                             path) is touched.

            -h, --help       Show usage information and exit.

            --ask-factory, --keep-factory and --no-factory are mutually
            exclusive.

        WHAT GETS RESTORED
            Default (no flags - overwrite everything):
                /bin/*.cs              Built-in user commands
                /sbin/*.cs             Built-in system/admin commands
                /lib/*.cs              Built-in libraries
                /usr/share/man/*.txt   Manual pages
                /etc/*                 Factory configuration files
                                       (overwritten)

            With --ask-factory:
                Same as default, but you confirm each existing /etc file.

            With --keep-factory:
                Same as default, but any factory file that already exists
                is left untouched.

            With --no-factory:
                /bin/*.cs              Built-in user commands
                /sbin/*.cs             Built-in system/admin commands
                /lib/*.cs              Built-in libraries
                /usr/share/man/*.txt   Manual pages
                (Nothing in /etc is touched at all.)

        WHAT DOES NOT GET AFFECTED
            * User home directories (/home/*, /root)
            * User data in /var, /tmp, /opt
            * Mounted archives
            * User accounts and groups (/etc/passwd, /etc/shadow,
              /etc/group) - these are managed separately
            * Custom scripts you have written in /bin or /sbin
              (they will be overwritten only if they have the same
              name as a factory builtin)

        EXAMPLES
            Full reinstall, overwrite all factory configs (default):
                # reinstall

            Reinstall but be asked about each existing /etc file:
                # reinstall --ask-factory
                The file '/etc/sandbox.conf' already exists (y=overwrite, n=ignore, a=overwrite all, q=ignore all)
                > y
                The file '/etc/telnetd.conf' already exists (y=overwrite, n=ignore, a=overwrite all, q=ignore all)
                > a
                  Installed 12 factory file(s) (3 new, 9 overwritten, 0 kept)

            Reinstall but keep all existing /etc edits:
                # reinstall --keep-factory

            Reinstall binaries only, leave /etc completely alone:
                # reinstall --no-factory
                # reinstall --skip-factory

        PERMISSIONS
            Only root can run this command. Attempting to run it as a
            regular user results in:
                reinstall: permission denied (must be root)

        EXIT STATUS
            0   Success - factory files restored.
            1   Error - permission denied, unknown option, conflicting
                flags, or internal failure.

        NOTES
            * The factory source files are embedded in the NetNIX
              executable and loaded from the Builtins/, SystemBuiltins/,
              Libs/, helpman/, and Factory/ directories in the NetNIX
              installation folder.
            * After reinstall, the filesystem is automatically saved
              (api.Save() is called internally).
            * This command does not require a system restart.
            * If you want to export a backup before reinstalling, use
              the export command to save your current filesystem state
              to a zip archive first.
            * The summary line "(N new, N overwritten, N kept)" reports
              what actually happened to factory files.
