        NAME
            reinstall - restore factory binaries, libraries, and man pages

        SYNOPSIS
            reinstall [--no-factory] [--skip-factory]
            reinstall -h | --help

        DESCRIPTION
            reinstall is a system maintenance command that overwrites the
            built-in NetNIX commands, libraries, and documentation with
            the factory versions shipped inside the NetNIX executable.

            This is useful when:
            * You have updated NetNIX and want to install new commands
            * A built-in command or library has been accidentally modified
            * Files in /bin, /sbin, or /lib have been deleted or corrupted
            * You want to restore the default manual pages

            By default, reinstall also restores factory configuration
            files stored in the Factory/ directory and installed to /etc.
            This includes files like:
                /etc/httpd.conf
                /etc/telnetd.conf
                /etc/nshrc

            To preserve your customized configuration files, use the
            --no-factory option.

        OPTIONS
            --no-factory     Skip reinstalling factory config files from
            --skip-factory   the /etc directory. Binaries, libraries, and
                             man pages are still restored, but your
                             existing /etc configuration is preserved.

            -h, --help       Show usage information and exit.

        WHAT GETS RESTORED
            Without --no-factory (default behavior):
                /bin/*.cs              Built-in user commands
                /sbin/*.cs             Built-in system/admin commands
                /lib/*.cs              Built-in libraries
                /usr/share/man/*.txt   Manual pages
                /etc/*                 Factory configuration files

            With --no-factory:
                /bin/*.cs              Built-in user commands
                /sbin/*.cs             Built-in system/admin commands
                /lib/*.cs              Built-in libraries
                /usr/share/man/*.txt   Manual pages
                (Config files in /etc are NOT touched)

        WHAT DOES NOT GET AFFECTED
            * User home directories (/home/*, /root)
            * User data in /var, /tmp, /opt
            * Mounted archives
            * User accounts and groups (/etc/passwd, /etc/shadow,
              /etc/group) — these are managed separately
            * Custom scripts you have written in /bin or /sbin
              (they will be overwritten only if they have the same
              name as a factory builtin)

        EXAMPLES
            Full reinstall (default):
                # reinstall

            Reinstall binaries only, preserve /etc configs:
                # reinstall --no-factory
                # reinstall --skip-factory

            After a NetNIX update, restore new commands but keep
            your customized httpd.conf and telnetd.conf:
                # reinstall --no-factory

        PERMISSIONS
            Only root can run this command. Attempting to run it as a
            regular user results in:
                reinstall: permission denied (must be root)

        EXIT STATUS
            0   Success - factory files restored.
            1   Error - permission denied, unknown option, or internal
                failure.

        NOTES
            * The factory source files are embedded in the NetNIX
              executable and loaded from the Builtins/, SystemBuiltins/,
              Libs/, helpman/, and Factory/ directories in the NetNIX
              installation folder.
            * After reinstall, the filesystem is automatically saved
              (api.Save() is called internally).
            * This command does not require a system restart.
            * If you want to export a backup before reinstalling, use
              the export command to save your current filesystem state
              to a zip archive first.

        SEE ALSO
