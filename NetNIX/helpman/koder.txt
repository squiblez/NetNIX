        NAME
            koder - AI-powered command generator for NetNIX

        SYNOPSIS
            koder
            koder --config
            koder --set <key> <value>
            koder --reset
            koder --help

        DESCRIPTION
            Koder uses a KoboldCpp AI model to generate complete, working
            NetNIX command scripts (.cs files) based on a natural language
            description. It loads a detailed system prompt from
            /etc/opt/koder/commandprompt.txt that teaches the AI about the
            NetNIX environment, the NixApi, sandbox restrictions, and
            coding conventions.

            The generation process is interactive:

            1. Koder connects to the KoboldCpp API
            2. You provide a command name (e.g. "massrename")
            3. You describe what the command should do
            4. Koder sends the description plus the system prompt to the AI
            5. The AI generates a complete .cs command file
            6. You preview the code and choose where to save it

            Because code generation can be complex, Koder uses a very high
            timeout (default: 15 minutes) and large token limits.

        SUBCOMMANDS
            koder
                Start the interactive generation wizard.

            koder --config
                Display current Koder settings and file paths.

            koder --set <key> <value>
                Persistently change a setting (stored per-user in
                ~/.config/koder.conf via settingslib).

            koder --reset
                Reset all Koder settings to defaults.

        SETTINGS
            All settings use app name "koder" in settingslib.

            Key             Default         Description
            endpoint        (from kobold)   KoboldCpp API URL
            max_length      4096            Max tokens to generate
            max_context     8192            Max context window (tokens)
            gen_timeout     3600            Generation timeout (seconds)
            temperature     0.3             Sampling temperature

            Notes:
            - Lower temperature (0.1-0.3) produces more deterministic,
              reliable code. Higher values add creativity but may cause
              syntax errors.
            - gen_timeout defaults to 3600 seconds (60 minutes). For very
              complex commands or slower hardware, increase this.
            - max_length of 4096 allows generating substantial commands.
              Increase if code is being cut off.
            - max_context of 8192 accommodates the large system prompt
              plus the user description and generated code.

        SYSTEM PROMPT
            The system prompt is loaded from:
                /etc/opt/koder/commandprompt.txt

            This file is installed from Factory files during first-run
            setup and can be restored with 'reinstall'.

            The prompt teaches the AI about:
            - NetNIX command structure (class layout, entry point)
            - The full NixApi surface (filesystem, users, networking, etc.)
            - Sandbox restrictions (blocked namespaces and tokens)
            - Coding conventions (arg parsing, help flags, error handling)
            - Example commands for reference

            Root users can edit the prompt to customize generation behavior:
                edit /etc/opt/koder/commandprompt.txt

        SETUP
            1. Install and run KoboldCpp with a capable code model
            2. Ensure the kobold endpoint is set:
               kobold --set endpoint http://HOST:PORT
               (or: koder --set endpoint http://HOST:PORT)
            3. Run: koder

        EXAMPLES
            # Generate a file renaming utility
            koder
            Command name: massrename
            Description> Recursively renames all files in a directory
                       > matching a pattern. Supports --dry-run flag.
                       > Usage: massrename <dir> <old-pattern> <new-pattern>
                       >

            # Generate and install as a global command (as root)
            koder
            (follow prompts, choose option 2 to install to /bin)

            # Adjust settings for a slow machine
            koder --set gen_timeout 1800
            koder --set max_length 8192

            # View current settings
            koder --config

        FILES
            /etc/opt/koder/commandprompt.txt   System prompt (factory file)
            ~/.config/koder.conf               Per-user settings
            /etc/opt/koder.conf                System-wide settings

        TIPS
            - Be specific in your description. Include:
              * What arguments/flags the command takes
              * What it should output
              * Error cases to handle
              * Example usage
            - The AI generates better code with detailed descriptions
            - Review generated code before installing to /bin
            - If the output is cut off, increase max_length
            - If generation times out, increase gen_timeout
            - Use a code-capable model (e.g. CodeLlama, DeepSeek-Coder)
